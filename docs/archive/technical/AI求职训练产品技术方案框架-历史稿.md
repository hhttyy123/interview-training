# AI 求职训练产品技术方案：实时语音面试版

## 一、产品目标与范围

产品聚焦“具体公司+具体岗位”的 AI 求职训练。首期核心不是文字问答或录音提交，而是用户与 AI 面试官进行接近真实面试节奏的**实时语音对话**：AI 当场提问，用户即时作答，AI 根据回答继续追问，面试结束后给出可解释反馈。

### 首期能力

- 上传简历并输入目标公司、目标岗位或岗位 JD。
- 获取个人经历与目标岗位之间的差距诊断。
- 基于岗位与个人经历生成训练重点。
- 与 AI 面试官进行实时语音对话：提问、作答、追问和应答连续进行。
- 面试结束后获取结构化反馈、改进建议和下一轮训练任务。
- 查看历史训练进步与岗位路径建议。

### 暂不优先

- 视频表情、肢体动作分析。
- 覆盖大量公司的真实内部面试题库。
- 真实岗位投递与招聘流程管理。
- 长期职业规划平台功能。

## 二、核心能力：实时语音面试

### 用户体验

- 用户点击开始后，直接进入语音面试房间。
- AI 以语音提出问题，用户无需按题提交录音，可自然连续作答。
- AI 能识别用户回答结束、根据内容即时追问，并控制整场面试节奏。
- 面试结束后，系统基于完整对话生成评分、依据与改进任务。

### 实时交互链路

```text
用户麦克风音频流
-> 实时传输
-> 语音识别 / 实时对话模型理解回答
-> AI 依据简历、JD 和上下文生成追问
-> 语音合成或原生语音输出
-> 用户听到下一轮问题
```

这一链路必须在面试进行过程中持续运行，而不是“录完一段音频后再统一分析”。系统同时保存会话音频、转写文本、追问轨迹和时间信息，供面试后的完整评估使用。

### 首期体验指标

| 指标 | 关注目标 |
| --- | --- |
| 语音响应延迟 | 用户回答结束后尽快听到下一问，避免明显停顿 |
| 打断与轮次判断 | 能判断用户是否说完，减少错误抢话或长时间等待 |
| 转写可用性 | 转写内容足以支持追问与事后评分 |
| 会话完成率 | 用户能顺利完成一整场语音模拟面试 |

## 三、用户流程

```text
注册/登录
-> 填写求职目标
-> 上传简历与目标岗位 JD
-> 获取岗位差距诊断
-> 选择面试场景和训练重点
-> 进入实时语音模拟面试
-> AI 根据回答进行连续追问
-> 获取评分与详细反馈
-> 更新能力画像和进步记录
-> 推荐下一步训练或岗位路径
```

## 四、核心模块

| 模块 | 主要能力 |
| --- | --- |
| 用户与权限 | 登录注册、资料、套餐和使用权限 |
| 简历与岗位解析 | 提取用户经历及 JD 的核心能力要求 |
| 差距诊断 | 对比个人经历与岗位要求，生成差距与优势说明 |
| 实时语音面试 | 语音输入输出、实时转写、多轮对话和动态追问 |
| 评分反馈 | 根据完整面试转写生成评分、依据和改进建议 |
| 进步追踪 | 保存历史表现，展示能力变化及复训建议 |
| 路径推荐 | 推荐主目标、相邻岗位和保底方向 |
| 付费服务 | 免费体验、套餐和使用次数控制 |

## 五、系统架构

```text
Web / 小程序面试端
        |
业务后端 API ------ PostgreSQL / 对象存储
        |
实时语音会话服务
        |---- FunASR Paraformer-zh-streaming：实时语音转写
        |---- 对话模型适配器：面试官提问与追问决策
        |---- 流式 TTS 适配器：AI 回答转语音
        |---- 会话状态：上下文、时长、题目与转写
        |
面试后评估服务 ---- JD / 简历 / 会话转写 / 评分标准
```

### 实时语音面试关键要求

- 支持低延迟语音流传输，避免每轮都需上传完整录音。
- 支持语音活动检测与轮次控制，判断用户何时结束回答。
- `FunASR Paraformer-zh-streaming` 持续生成用户回答转写。
- 对话模型以流式文本生成追问，并交由流式 `TTS` 尽早播放，减少等待感。
- 保存完整音频与转写，评分在面试完成后基于完整上下文生成。
- 弱网或服务异常时支持中断恢复，并明确提示用户录音与转写状态。

## 六、核心数据

| 数据对象 | 说明 |
| --- | --- |
| `User` | 用户账号、基础信息和权限 |
| `Resume` | 用户上传的简历及解析结果 |
| `TargetPosition` | 目标公司、岗位、JD 与岗位画像 |
| `AbilityProfile` | 用户能力画像、优势与短板标签 |
| `InterviewSession` | 一次实时语音面试的场景、状态与结果 |
| `InterviewTurn` | 每轮问题、回答、追问和时间戳 |
| `Transcript` | 实时语音转写及校正文本 |
| `AudioAsset` | 面试录音文件及访问权限 |
| `Feedback` | 评分维度、证据、问题与改进建议 |
| `ProgressSnapshot` | 阶段性能力变化与训练结果 |
| `RecommendedPath` | 主目标、相邻岗位与保底路径建议 |

评分、依据和进步变化必须结构化存储，才能展示复训效果并校验 AI 判断质量。

## 七、AI 与语音能力

### 会话前

- 解析简历、岗位 JD 和目标公司公开信息。
- 生成本轮面试考察维度、主问题和追问策略。

### 会话中

- 语音通道持续传输音频，并识别用户开始与结束作答的时机。
- `FunASR Paraformer-zh-streaming` 将语音流转换为增量文本，由对话模型结合简历、JD 和上下文决定追问、澄清或进入下一题。
- 流式 `TTS` 将 AI 问题边生成边播报给用户。
- 会话控制限制面试时长、问题数量和无关对话偏移。

### 会话后

- 基于完整转写、JD 和评分标准生成反馈。
- 标注得分依据对应的回答片段或缺失证据。
- 更新能力画像、训练重点和下一轮面试建议。

### 反馈输出示例

```json
{
  "overall_score": 72,
  "dimensions": {
    "回答结构": 75,
    "经历证据": 60,
    "岗位理解": 70,
    "临场应对": 73
  },
  "strengths": [
    "项目背景描述完整"
  ],
  "problems": [
    "项目成果缺少量化证据"
  ],
  "next_actions": [
    "使用 STAR 结构重写项目回答",
    "补充项目结果指标并重新作答"
  ]
}
```

### 可信度要求

- 评分需关联面试转写中的依据或缺失点。
- 无充分信息时应提示用户补充内容，而不是强行给出结论。
- 同岗位、同场景采用稳定的评分标准。
- 不能声称掌握某家公司未公开的内部标准。
- 不能承诺用户获得 offer 的确定概率。

## 八、数据来源与合规

### 信息来源

- 用户主动上传或粘贴的岗位 JD。
- 企业官网公开招聘信息。
- 自建通用岗位能力标签库。
- 经用户授权后的匿名化训练数据。

### 隐私要求

- 简历和面试录音需加密存储并限制访问。
- 录音前明确征得同意，提供删除个人数据的入口。
- 数据用于训练或分析前必须取得授权并进行匿名化。
- 反馈不能表述为录用保证，公开信息需说明来源边界。

## 九、技术选型参考

| 能力 | 可选技术方案 |
| --- | --- |
| 面试端 | Web 应用或微信小程序，支持麦克风与流式播放 |
| 后端服务 | Python + FastAPI，或 Node.js 服务框架 |
| 业务数据库 | PostgreSQL |
| 文件存储 | 对象存储，用于简历、面试录音和报告 |
| 实时通信 | 浏览器至本地服务优先使用 WebSocket 音频流 |
| 实时识别 | `FunASR Paraformer-zh-streaming` |
| 对话能力 | 可替换的流式对话模型适配器 |
| 语音输出 | 可替换的流式 `TTS` 适配器 |
| 岗位检索 | JD 数据库与岗位标签体系 |
| 部署 | 云服务器或 Serverless 服务 |

## 十、首期开发顺序

1. 跑通浏览器音频采集与 `FunASR` 实时转写，并验证轮次和打断检测。
2. 接入流式对话模型与流式 `TTS`，完成可打断的双向语音对话。
3. 接入简历/JD，使实时追问能够围绕目标岗位与用户经历展开。
4. 实现会话后结构化反馈与历史表现记录。
5. 增加路径推荐、免费体验与付费验证。

## 十一、关键指标与风险

| 关注点 | 需要验证的内容 |
| --- | --- |
| 实时体验 | 对话延迟、识别准确率和中断恢复是否可接受 |
| AI 反馈质量 | 评分是否稳定，解释和建议是否被认可 |
| 用户留存 | 用户是否愿意多次训练，而非只看一次报告 |
| 训练效果 | 训练得分提升是否能对应真实面试表现改善 |
| 使用成本 | 本地推理资源、对话模型与流式 TTS 的成本是否可控 |
| 数据合规 | 简历与面试录音的授权、存储、删除是否合规 |
| 商业模式 | 免费体验与中低价格套餐能否支撑转化 |

关键验证指标可包括：语音首响延迟、转写准确体验评价、完整面试完成率、复训率、反馈有用性评分和付费转化率。

## 十二、总结

```text
用户简历与目标岗位信息
-> 岗位差距诊断
-> 实时语音模拟面试与动态追问
-> 可解释反馈
-> 能力画像与进步记录更新
-> 下一轮训练及岗位路径推荐
```

首期最关键的是验证两件事：用户是否认可结合目标岗位的实时语音面试体验，以及详细反馈能否推动用户持续复训。
