# 验证 V0 延迟与持续对话可用性

Status: `ready-for-human`
Type: `HITL`

## What to build

以固定的对话场景实际体验完整实时语音链路，记录识别、回复、播报和打断的延迟与问题归属，形成是否继续进入求职面试场景开发的判断。

## Acceptance criteria

- [ ] 准备覆盖短句、长句、中英文混合和主动打断的测试对话清单。
- [ ] 完成至少一次 5 分钟以上的连续双向语音体验。
- [ ] 记录首字转写延迟、终句确认等待、AI 首字回复延迟、AI 首音频延迟和打断停止延迟。
- [ ] 记录识别准确度、语音播放与对话自然度方面的主要问题，并标明归属组件。
- [ ] 验证当前版本未持久化音频和转写内容。
- [ ] 形成继续优化组合链路、引入云端端到端对照，或进入面试场景开发的结论。
