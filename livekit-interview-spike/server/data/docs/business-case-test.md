# 业务 Case 拆解测试资料

业务、产品、运营、咨询和数据分析岗位，经常需要通过 Case 追问来判断候选人的结构化拆解能力。Case 追问不只是让候选人给一个方案，而是检查他如何定义问题、建立假设、权衡资源、设计验证和识别风险。

## 一、目标定义

优秀候选人不会一上来就给方案，而会先确认目标。例如“提升用户活跃”可能有很多含义：提升 DAU、提升人均使用时长、提升次日留存、提升核心功能使用率，还是提升付费转化。目标不同，方案和指标完全不同。

面试官可以追问：

- 你会先确认哪个核心目标？
- 这个目标为什么比其他目标更重要？
- 如果只能选一个北极星指标，你会选什么？
- 这个指标有没有副作用？

## 二、用户或客户分层

很多候选人的方案听起来正确，但没有区分用户类型。业务问题通常不是所有用户都一样。新用户、老用户、高价值用户、流失风险用户、企业客户、个人客户，在动机、阻力和转化路径上可能完全不同。

面试官可以追问：

- 你会先看哪一类用户？
- 不同用户群体的核心阻力有什么差异？
- 你怎么判断这个群体值得优先解决？
- 有没有一类用户不应该优先优化？

## 三、约束与资源

真实业务里方案受时间、成本、技术、合规、组织协作和渠道资源限制。候选人如果只讲理想方案，没有讲约束，说明方案落地能力不足。

面试官可以追问：

- 如果只有两周时间，你会先做什么？
- 如果研发资源不足，你怎么调整方案？
- 这个方案有没有合规或品牌风险？
- 哪些动作可以低成本验证？

## 四、方案选项与优先级

好的 Case 回答通常会提出多个方案，并说明优先级依据。优先级可以基于影响面、确定性、成本、风险、学习价值或战略匹配度。

面试官可以追问：

- 你至少会比较哪三个方案？
- 为什么第一个做这个，而不是另一个？
- 你会用什么标准排序？
- 如果第一个方案失败，第二选择是什么？

## 五、指标体系与验证

业务方案必须能验证。指标不应只看最终结果，也要看过程指标和护栏指标。例如优化转化率时，要同时关注转化、留存、投诉、退款、用户满意度和长期价值。

面试官可以追问：

- 你会设置哪些主指标和辅助指标？
- 指标口径怎么定义？
- 多久看一次结果？
- 怎么判断结果不是偶然波动？
- 有没有需要防止被优化坏的护栏指标？

## 六、风险与复盘

Case 面试不仅看候选人会不会设计方案，也看他是否能预判失败。优秀候选人通常能说出方案失败的可能原因，以及如何提前监控。

面试官可以追问：

- 这个方案最可能失败在哪里？
- 你会提前监控哪些风险信号？
- 如果结果不达预期，你怎么定位原因？
- 下一轮迭代会怎么做？

## 七、反模式

不要把 Case 变成脑筋急转弯。业务拆解的核心不是猜面试官心里的标准答案，而是展示清晰的假设、取舍和验证路径。

不要只听候选人讲方案。面试官必须追问目标、用户、约束、指标和风险，否则无法判断候选人的业务判断质量。

不要过度依赖固定行业知识。即使候选人不了解某个具体行业，也可以通过结构化提问展示问题拆解能力。
